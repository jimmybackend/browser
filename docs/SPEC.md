- No inventar tablas ni campos.
- Usar PDO con consultas preparadas.
- No exponer errores internos al usuario final.
- No subir secretos.
- Mantener estructura existente.
- Documentar cambios importantes.

## Criterios de aceptación generales

El proyecto puede considerarse listo para revisión cuando:

- La aplicación puede instalarse siguiendo el README.
- La configuración necesaria está documentada.
- No hay errores fatales de PHP.
- Las rutas principales funcionan.
- Las consultas críticas usan prepared statements.
- Las entradas del usuario se validan.
- Las salidas HTML se escapan cuando aplica.
- Las rutas privadas están protegidas.
- El CI pasa correctamente.
- Los pendientes están documentados.
