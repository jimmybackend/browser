# Tareas del proyecto

## Fase 1: Diagnóstico

- [ ] Revisar estructura del repositorio.
- [ ] Identificar versión de PHP.
- [ ] Identificar si usa Composer.
- [ ] Identificar framework o arquitectura.
- [ ] Identificar punto de entrada de la aplicación.
- [ ] Identificar configuración requerida.
- [ ] Identificar dependencias.
- [ ] Identificar base de datos requerida.
- [ ] Identificar archivos sensibles.
- [ ] Crear reporte inicial de estado.

## Fase 2: Instalación

- [ ] Crear o actualizar README.
- [ ] Crear o actualizar `.env.example` si aplica.
- [ ] Documentar variables de entorno.
- [ ] Documentar instalación local.
- [ ] Documentar comandos de validación.
- [ ] Verificar conexión a base de datos.

## Fase 3: Funcionalidad base

- [ ] Revisar login.
- [ ] Revisar logout.
- [ ] Revisar sesiones.
- [ ] Revisar rutas protegidas.
- [ ] Revisar navegación principal.
- [ ] Revisar formularios principales.
- [ ] Revisar CRUD principal.
- [ ] Revisar búsqueda/listado si aplica.
- [ ] Revisar mensajes de error y éxito.

## Fase 4: Seguridad

- [ ] Revisar SQL injection.
- [ ] Revisar XSS.
- [ ] Revisar CSRF en formularios sensibles.
- [ ] Revisar autenticación.
- [ ] Revisar autorización.
- [ ] Revisar manejo de contraseñas.
- [ ] Revisar exposición de errores.
- [ ] Revisar archivos sensibles.
- [ ] Revisar logs.

## Fase 5: Pruebas y validación

- [ ] Agregar validación de sintaxis PHP.
- [ ] Agregar pruebas mínimas si el proyecto tiene estructura para pruebas.
- [ ] Agregar PHPStan si aplica.
- [ ] Agregar PHPUnit si aplica.
- [ ] Agregar workflow de GitHub Actions.
- [ ] Documentar pruebas manuales.

## Fase 6: Finalización

- [ ] Limpiar código muerto.
- [ ] Ordenar documentación.
- [ ] Crear checklist de producción.
- [ ] Documentar despliegue.
- [ ] Documentar riesgos.
- [ ] Dejar PR listo para revisión.
